INTRO                                                          *orchestrate-intro*

orchestrate.nvim is an Agent Orchestration Workspace for Neovim.

It is not a chat UI and it is not a terminal wrapper.

Core flow:

    ACP -> Store -> Renderer -> Neovim Buffers/Windows

Key points:

- Three buffers: Browse / Todo / Input
- Input buffer uses acwrite and sends on :w
- State changes go through actions
- Renderers only render session state
- Default transport is Claude Code CLI
- Mock transport is still available for development and tests

==============================================================================
COMMANDS                                                    *orchestrate-commands*

:OrchestrateOpen
    Open the workspace.

:OrchestrateClose
    Close the workspace.

:OrchestrateSend [text]
    Send text directly, or send the current Input buffer when no argument is
    provided.

:OrchestrateResume [text]
    Resume the current Claude Code session using the stored session id.

:OrchestrateContinue [text]
    Continue the latest Claude Code session in the current working directory.

CONFIG                                                        *orchestrate-config*

Default config:

>lua
  require("orchestrate").setup({
    layout = {
      browse_width = 0.7,
      todo_height = 0.5,
      min_browse_width = 40,
      min_sidebar_height = 6,
    },
    ui = {
      input_filetype = "markdown",
    },
    transport = {
      name = "claude_code",
      claude_code = {
        command = "claude",
        resume_strategy = "session_id",
        fallback_to_mock = false,
      },
    },
    mock = {
      chunk_delay = 160,
    },
  })
<

==============================================================================
USAGE                                                          *orchestrate-usage*

1. Run :OrchestrateOpen
2. Type into the Input buffer
3. Run :w
4. Watch Browse for streaming output and Todo for structured updates

Run |:checkhealth| orchestrate to verify Claude Code availability.
